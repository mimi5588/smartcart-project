# SmartCart - הסבר והפעלה

## מה זה הפרויקט?

SmartCart הוא פרויקט Frontend שנבנה עם React ו-Vite. המטרה שלו היא לשמש בסיס לאפליקציית קניות חכמה: מסך בית למשתמש, עמוד סריקת מוצרים, דשבורד נתוני קניות וחיסכון, ועמוד פרופיל משתמש.

נכון למצב הקוד הנוכחי, רוב המסכים הם שלד ראשוני ולא מערכת מלאה שמחוברת לשרת או למסד נתונים.

## הטכנולוגיות המרכזיות

- React - בניית ממשק המשתמש.
- Vite - סביבת פיתוח ובנייה מהירה לפרויקט React.
- React Router - ניווט בין עמודים באפליקציה.
- ESLint - בדיקות סגנון ובעיות JavaScript/React.

## הסבר על הקבצים החשובים

### `package.json`

מגדיר את שם הפרויקט, התלויות והפקודות שאפשר להריץ.

הפקודות המרכזיות:

```bash
npm run dev
npm run build
npm run lint
npm run preview
```

### `src/main.jsx`

נקודת הכניסה של האפליקציה. הקובץ מחבר את React לאלמנט `root` שבתוך `index.html` ומרנדר את `App`.

### `src/App.jsx`

מגדיר את הראוטר של האפליקציה:

- `/` - עמוד הבית.
- `/scan` - עמוד סריקת מוצרים.
- `/dashboard` - עמוד דשבורד.
- `/profile` - עמוד פרופיל.

### `src/pages`

תיקיית עמודי האפליקציה:

- `HomePage.jsx` - עמוד הבית עם ברכת משתמש, תקציב חודשי וכפתור התחלת קנייה.
- `ScanPage.jsx` - עמוד בסיסי לסריקת מוצרים.
- `DashboardPage.jsx` - עמוד בסיסי לסטטיסטיקות קנייה וחיסכון.
- `ProfilePage.jsx` - עמוד בסיסי למידע והגדרות משתמש.
- `ShoppingSetupPage.jsx` - קובץ קיים אך כרגע ריק.

### `src/components`

רכיבים משותפים:

- `Navbar.jsx` - פס ניווט עליון.
- `Footer.jsx` - פוטר תחתון.

### `DESIGN.md`

מסמך עיצוב שמגדיר צבעים, טיפוגרפיה, ריווחים, כפתורים, כרטיסים וניווט.

## איך מפעילים את הפרויקט?

### 1. כניסה לתיקיית הפרויקט

ב-PowerShell:

```powershell
cd "C:\Users\mayco\OneDrive\שולחן העבודה\חשוב\CODEX פרויקטים\smartcart\smartcart"
```

שימו לב: קיימת תיקיית `smartcart` חיצונית ובתוכה תיקיית `smartcart` נוספת. לפי המבנה המקומי, הפרויקט שרץ בפועל נמצא בתיקייה הפנימית.

### 2. התקנת תלויות

אם `node_modules` כבר קיים, יכול להיות שאין צורך להריץ את זה שוב. אם מתחילים מסביבה נקייה:

```powershell
npm install
```

### 3. הרצה במצב פיתוח

```powershell
npm run dev
```

לאחר ההרצה Vite יציג כתובת מקומית, בדרך כלל:

```text
http://localhost:5173/
```

פותחים את הכתובת בדפדפן.

### 4. בדיקת Build

כדי לבדוק שהפרויקט יכול להיבנות לגרסת Production:

```powershell
npm run build
```

אם הכל תקין, תיווצר תיקיית `dist`.

### 5. תצוגה מקדימה של Build

אחרי `npm run build`, אפשר להריץ:

```powershell
npm run preview
```

גם כאן Vite יציג כתובת מקומית לפתיחה בדפדפן.

### 6. בדיקת Lint

```powershell
npm run lint
```

הפקודה בודקת בעיות סגנון וקוד לפי הגדרת ESLint.

## מצב נוכחי ודגשים

- אין Backend בפרויקט כרגע.
- אין מסד נתונים.
- אין מערכת משתמשים אמיתית או התחברות.
- עמוד הסריקה הוא Placeholder בלבד, כלומר עדיין אין סריקת ברקוד אמיתית.
- הניווט ב-`Navbar` מוצג כטקסטים רגילים ולא כקישורים פעילים.
- חלק מהטקסטים בעברית נראים מקודדים בצורה שבורה, כנראה בגלל בעיית Encoding בקבצים.
- קיימת כפילות תיקיות: בתוך הפרויקט יש עוד תיקיית `smartcart` שמכילה עותק נוסף של הפרויקט. כדאי לנקות את זה בהמשך כדי למנוע בלבול.

## איך ממשיכים לפתח?

צעדים הגיוניים להמשך:

1. לתקן את בעיות הקידוד בעברית.
2. להפוך את פריטי הניווט ב-`Navbar` לקישורים עם `Link` מתוך `react-router-dom`.
3. להשלים את `ShoppingSetupPage.jsx` או למחוק אותו אם לא משתמשים בו.
4. לבנות מסך סריקה אמיתי, למשל עם ספריית סריקת ברקודים.
5. להוסיף ניהול מצב לרשימת קניות, תקציב וסיכומי קנייה.
6. להחליט אם צריך Backend, ואם כן להוסיף API ומסד נתונים.

## סיכום קצר

SmartCart הוא בסיס לאפליקציית קניות חכמה ב-React. כרגע הוא מתאים להצגה ראשונית, ניסוי בעיצוב, והמשך פיתוח של עמודים ופיצ'רים. כדי להריץ אותו צריך להיכנס לתיקייה הפנימית של הפרויקט, להתקין תלויות עם `npm install`, ואז להריץ `npm run dev`.
